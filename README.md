## **Overview**  
This repository is a step-by-step guide for beginners to learn the process of:  
- Creating a new repository on GitHub.  
- Cloning the repository locally using GitHub Desktop.  
- Adding and committing changes.  
- Pushing changes to GitHub.  

---

## **Prerequisites**  
Before starting, ensure you have the following installed and configured:  
- [GitHub Desktop](https://desktop.github.com/)  
- A [GitHub](https://github.com/) account.  
- Basic knowledge of file navigation on your operating system.  

---

## **Steps to Push Code**  
### **1. Create a New Repository on GitHub**  
1. Log in to GitHub.  
2. Click the **+** icon and select **New repository**.  
3. Provide a repository name and click **Create repository**.  

### **2. Clone the Repository Using GitHub Desktop**  
1. Open GitHub Desktop.  
2. Go to **File > Clone Repository** and select your new repository.  

### **3. Add Your Code**  
1. Copy your project files into the cloned repository folder.  

### **4. Commit Your Changes**  
1. In GitHub Desktop, add a commit message and click **Commit to `main`**.  

### **5. Push Your Changes**  
1. Click **Push origin** to upload your code to GitHub.  

---

## **Common Issues and Fixes**  
### **Error: Cannot Push Unborn Head**  
This occurs when no commits exist in the repository. Ensure you commit your changes before pushing.  

### **Error: Lock File Already Exists**  
Delete the `index.lock` file from the `.git` directory to resolve this.  

### **Other Errors**  
Refer to the [GitHub Desktop Troubleshooting Guide](https://docs.github.com/en/desktop/troubleshooting) for more information.
